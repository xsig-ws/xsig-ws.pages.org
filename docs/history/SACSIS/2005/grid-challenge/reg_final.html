GCF環境での決勝のプログラムのコンパイルとリンク方法

決勝の出題APIは、3月7日 17:00ごろに、各クラスタのコンパイルノードの

C言語版
  • ~tau/local/Linux/include/grid_challenge_api.h (ヘッダ)
  • ~tau/local/Linux/lib/libgrch.a (ライブラリ)
  • ~tau/local/Linux/lib/libcrypto.a (暗号化ライブラリ)
Java版
  • ~tau/local/Linux/lib/grch.jar

に置かれる。参加者はこのAPIをリンク(C言語版)、 もしくはclasspassを設定(Java版)して用いること。

決勝用ライブラリは、予選用ライブラリを上書きする形で、 同じディレクトリに置かれる。 決勝の空き時間に、予選問題、お試し版の問題を使って プログラムのテストを行なう際には、 この決勝用ライブラリを用いて良い。 ただし、決勝用の問題での実行は空き時間には行なわないこと。

C言語版におけるlibcrypto.aはOpenSSLに付属のものであるため、 標準的なパス(/usr/libや/usr/local/ssl/libなど)にもすでにインストール されている可能性がある。その場合はどちらを利用しても構わない。

解答の提出先と方法

規定課題決勝は初級部門と上級部門の2つがある。 初級部門の問題番号は21〜23、上級部門は21〜26である。

参加者は、それぞれの問題の正解時にライブラリが出力する証明文、

[Grid Challenge] << Correct, prob 21, 3.0sec, at Mon Mar 7 21:13:55 2005. / key = jjbmanchhejjhkjgbeplgehkpnenljba >>

のように"<< "と">> "でくくられた出力行を、 解答すべき全ての問題(初級は3問、上級は6問)の、 全ての実行(各問題毎に2回ずつ)について集め、 grid-challenge-secretary@logos.ic.i.u-tokyo.ac.jpまで メールで提出しなければならない。 提出の際は、各証明文に余分な改行や編集が入らないよう、 添付テキストファイルとしてメールすること。

また、解答に用いたプログラムのソース一式も同様に添付し、 提出しなければならない。 上級部門への挑戦時、 初級と上級でプログラムが異なる場合は両者ともに提出すること。 また、標準のGCF環境にない、 一般に公開されているライブラリ等を各自でインストールして用いている場合は、 そのライブラリまで提出する必要はない。 ただし、委員会が後ほど問い合わせを行なった時に 解答に用いたバージョンが確実に手に入るよう、 ライブラリは各自で保管しておくこと。 (以上、プログラムソースの提出方法について3/11に追加)

メールの例は以下の通り。 

To: grid-challenge-secretary@logos.ic.i.u-tokyo.ac.jp
Subject: grch000 初級 解答提出
---
チーム名: grch000
初級部門の解答です。

<以下はインラインではなく添付文書とする>
[Grid Challenge] << Correct, prob 21, 5.0sec, at Mon Mar  7 21:13:55 2005. / key = bmjjanchhejjhkjgbeplgehkpnenljba >>
[Grid Challenge] << Correct, prob 21, 4.2sec, at Mon Mar  7 21:19:57 2005. / key = jjbmaneplgehkpnenlchhejjhkjgbjba >>
…
[Grid Challenge] << Correct, prob 23, 3.8sec, at Mon Mar  7 22:07:37 2005. / key = neenjjbmalchplgehkpnhejjhkjgbjba >>

<さらに、解答に用いたプログラムを圧縮し、以下に添付する>

結果提出はなるべく速やかに、 最後の結果が出てから遅くとも一日以内に行なうこと。 上級部門の6問を正解した参加者は、 6問分の正解証明文を集めたメールと、初級用の3問の正解証明文を集めたメールの つごう2通を別々に提出すること。

また、参加者は提出した結果の実行の様子(連続して2回成功する様子)を収めた 画面の動画キャプチャをとり、委員会に提出すること。 提出方法は追って連絡する。 提出の際は、各自適当なWebサーバにファイルを置き、URLを grid-challenge-secretary@logos.ic.i.u-tokyo.ac.jpまでメールで知らせること。 手近にWebサーバがない場合は、 istbs000.i.u-tokyo.ac.jpの各ユーザのホームディレクトリ下に適当に置き、 場所を同様にメールで知らせること。 (以上、提出方法について3/10に修正) 「連続実行」はそれぞれの問題ごとの2回の実行が連続していれば良く、 別々の問題に関して連続している必要はない。 動画キャプチャについても、それぞれの問題ごとに別れていても構わない。 キャプチャ方法については 決勝スロット割り当て の記述を参考にすること。

なお、初級部門と上級部門の両方へ挑戦しても良い。 挑戦する部門に関して事前の申請は必要ない。 両部門に挑戦する場合、問題番号21〜23は同じ問題を解答することになるが、 提出する実行結果は初級と上級で同じものでよい。

決勝の順位決定方法

初級、上級それぞれの部門において、以下の通り順位づけを行なう。 全ての問題を解く事ができなかった場合は失格とする。

  • 参加者は、各部門の全ての問題ごとに連続2回の解答試行を成功させなければならない。 2回の実行時間のうち短い方をその問題の所要時間とする。

  • 各問題について、全参加者中で3番目に短い所要時間を1とし、 それぞれの参加者の所要時間をそれとの比で表したものを、 その問題の「ペナルティ」とする。 ただし、1番短い所要時間であった参加者のみ特別にペナルティ0とする。 例えば、ある問題について4者の所要時間が A:"0:03:50"、B:"0:04:00"、C:"0:05:00"、D:"0:10:00"であった時、 4者のこの問題のペナルティはA:0.0、B:0.8、C:1.0、D:2.0となる。

    また、問題を解いた参加者が2名しかいなかった場合ペナルティは0.0と1.0とし、 1名しか解けなかった場合のペナルティは0.0とする。

  • 全ての問題のペナルティの和をとり、総ペナルティのより少ない参加者が上位となる。
