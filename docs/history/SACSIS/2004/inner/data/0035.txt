１．SACSIS2003 総括
(1) 総括報告
  ・準備〜開催の主要イベント一覧 ＜資料１＞
    - 2003年度は組織委員会を5回設けた
      o SACSIS初年度でありCFP作成まで時間がかかった
        （8月:Preliminary CFP, 11月:正式CFP）
      o 特に7, 8月はプログラム、キーワード作成で大変だった
      -> 2004年度は1回ぐらい省いても良いかも
    - メールは山のように飛び交った

(2) 論文募集・査読・採択 ＜資料２, ３＞
  ・call for paper
    - 新しい分野の論文を集めるため、キーワードを一生懸命考えた
      o 大項目（先進的計算システム、先進的計算システムを支える基盤技術、
        実用的基盤システム）を決めた後、各キーワードを決定。
    - レギュラー、ポスターとも論文フォーマットの説明が不足していた
    - プログラム担当とWeb担当の距離が離れている
      -> 広報委員長もPCメンバに入る
    - CFPの分野は今回ので特に問題なし。2004年度もこのままでOK

  ・PC/外部差読者構成、査読負荷
    - まずはPCメンバ（計37名）を召集、その後PCメンバに3名ずつ査読協力者を
      推薦してもらった
    - 割り当て会議(2/5)にて論文1本につき3名ずつ査読者を割り当て
      o 3名のうち1名は必ずPCメンバとし、またACS論文誌のメタレビューアに据えた
    - 採択会議(3/14)の出席者が少なかった（11名欠席）
      o 当落線上の論文担当のPCメンバが欠席したため、結構もめた
      o 最優秀学生論文賞が決められなかったことにも影響
      -> PCメンバ就任依頼時に予め採択会議日を決定しておくとか必要かも
      -> システムを変更しても解消されない？

  ・査読トラック
    - 従来通りトラックに分けて採録論文を決定
      o できれば全体で制御できないか？
    - どの分野にも割り振りづらい論文をどうするか、今後の課題
    - 査読協力者の専門分野・連絡先をきちんと把握しておくべきだった
    - どのようにPCメンバを割り当てるのが良いか？
      o JSPPのころは研究会選出PCメンバとトラックがほぼ一致していた
      o 新しい分野をどう分けるかが大変
      -> このトラックに対して何名出すように指定する？
      -> 研究会からPCメンバを選出⇒トラックへの割り付け
            ⇒割り当て会議、採録会議への名指しでの参加依頼、とするのはどうか？
      -> 割り当て会議、採録会議に出席できる人を出してもらう

  ・投稿数/採択数
    - 投稿79件（うち1件取り下げ）、そのうち43件を採録（採択率55.1%）
    - 「新しい風」は8/16(50%)、採択論文に占める割合は8/43(18.6%)
      -> SACSISの方向性は出せたと思う

  ・論文〆切
    - チキン・レースになることは避けなければいけない
      -> 登録〆切（ソフト・リミット）⇒登録〆切（ハード・リミット）
            ⇒論文アップロード（ハード・リミット）の三段階に分けるべき
      -> ポスター論文はsubmitを一回にすることが重要
    - 投稿〆切（１月末）は卒論・修論と重なっており大変
      １ヶ月遅らせたほうがいいかも
      -> さらに１ヶ月遅らせてSWoPPと一緒にする？
      -> 検討するとしても2006年度から

  ・ACSとの同時投稿
    - ページ数制限の問題
      o (参考)7月のCSシンポジウムは標準8ページ、最大10ページ
    - PCメンバの査読手順
      o ACSではメタ・レビューアとなるので、SACSISとACSでは正しくは
        査読手順が異なる。しかし、これを真面目に守ろうとすると〆切が
        厳しくなる

  ・その他
    - スケジュール的には計画通り進んだ
    - 査読する側のクオリティの問題
      o 査読者への説明不足、また漫然と査読している
      -> 査読報告フォーマットの見直しが必要
      -> 「プログラム委員」の手引きを作るべき
    - short paper が必ずしも有効に活用されていない
    - 一般向け論文賞があっても良いのでは？
      -> 企業は業績に反映する
      -> 教育機関、公的機関も同様
    - SACSIS単独投稿の人にLaTeXスタイルファイルを使えと強制するのは避けたい
      -> 英語の人、詰め込みたい人がかわいそう
    - 原稿執筆、講演依頼もメールで行う
      -> 採録になった人全員にメールを送付するシステムを作成中（中島）

   ☆ 最優秀学生論文賞、一般向け論文賞、short paper の長所・短所を
      リストアップして次回議論する

(3) 開催報告
  ・開催申請＜資料４＞
    - 開催申請を出したのが12月と遅かった
      o 理事会で承認されるのは年明け（１月中旬）になってしまい、
        そうなると他の手続きにも支障をきたす
      -> 10月ぐらいには出したほうが良い（担当：松原）

  ・call for participation＜資料５＞
    - SWoPP，DB Japan，各研究会のMLにCFPPを出した
    - CFPPを出すのが遅かった（４月）
      -> 年末あたりに出せるようにする、そのためには開催申請を早めに出す
    - 参加申し込みページが数日間閉鎖されてしまった
      o Webページを管理していた産総研のサーバの物理的な置き換えがあったから
        例年ならば土日の停止程度で済む
      o IPSJ のページから辿ってもらうことで対処
      o URL を直接指定できないので、IPSJのページに直接飛ぶのは難しい
    - 懇親会のことを書くのを忘れた
      o 当日配ったプログラムにも書き忘れてしまった

  ・プログラム＜資料６＞
    - 場所が東京だったので、三日目も遅くまでやった
      -> 2004年度は北海道だが、16:30ぐらいまでやっても問題なかろう
      -> 前日泊まってもらって初日午前中から開始、
         そして最終日は早めに切り上げるとかでもいいかも
    - 招待講演を何日目にするのかなかなか決まらなかった
      o イラク戦争が原因でHP社から海外出張禁止命令が出されていたため
      o 結局三日目に行なった（例年は二日目）
      -> 二日目はほかにもチュートリアルやポスターもあるので今回のように
         分けたほうがいいかも（集客の面でもメリットあり）
    - short paper
      o 発表は２０分とした
      o 今回は楽にセッション枠を決めることができた
    - 所属の略称
      o 知る手段がなかったので、Webを利用して１つ１つ調べた
      -> 統一性がなくてもいい、著者が思う略称を優先すべき
      -> 登録時に著者に書いてもらうようにする

  ・チュートリアル＜資料７＞
    - 両方とも100人ずつぐらい人が入り、まずまずだった
    - 講演者への謝金及び参加登録費（参加登録する必要はない）に関しては
      依頼する際に伝えておくべき
    - チュートリアルの座長は早めに決めておいたほうが良い
      o 今年は直前になって慌てて決めた
    - チュートリアルが別冊なのは？
      -> 締め切りが違うから
    - 論文予稿集と大きさが違うのは？
      -> 前年度のを踏襲
      -> PPTをB5で出すのは読みづらい
      -> 拡大縮小すると印刷代が高くつく
    - 予稿集をやめてCD-ROM だけにする？
      -> CD-ROMだけだと聴講時に見れないなどのデメリットがある
         別に用意するならOK

  ・ポスター展示＜資料８＞
    - 展示スペースがやはり狭かった
      -> 今回は約100m^2。150m^2ぐらいあると良い
    - パネル代が高かったこともあり、経費を抑えるために２日から1日に短縮
    - 今回デモは少なかったが、ちゃんとしたデモをするなら電源・ネットワークの
      確保をしっかり考えなくてはならない
    - SACSISに合わせたポスター展示のあり方を考えるべき
    - 論文形式も明確に規定したほうが良い

  ・会場関係報告（部屋割，break） ＜資料９＞
    - 電通大、早大、お茶大、東工大の学生（12名）にアルバイトを依頼
      -> 2004年度は学生の確保が大変だと思われるので、早めに手をうって
         おく必要がある
    - 無線LANがあまりうまくいかなかった
      o A会場の前のほうが届かなかった（会場中はサービスしてない、ロビーのみ）
    - break用のコーヒー・クッキー・ドーナツの量はちょうど良かった
      -> 水とかも用意したほうが良い
    - 運営委員会用の会議スペースも確保するのを忘れずに

  ・参加者統計 ＜資料１０＞
    - 相変わらず企業からの参加が少ない
    - 来年は全体的に減りそうなので、何か工夫しなければいけない

  ・参加者アンケート ＜資料１１＞
    - Gridに興味を持っていた人が多かった
    - SACSISの目標はある程度達成されたといっていいかも
      o 多くの人がSACSISになって変わったと認識してくれた
